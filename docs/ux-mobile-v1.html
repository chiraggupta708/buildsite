📱 BuildSite v1 — Mobile UX

Phone-first design. Every screen optimized for thumb-reach, one-handed use, and quick data entry on-site.

Core philosophy: A contractor on-site uses their phone one-handed, often in sunlight, with dusty hands. Every screen must be thumb-friendly, glanceable, and require 0 training. Bottom nav replaces sidebar. Big touch targets (min 44px). No hover states. No grids that need pinch-zoom.

📋 Mobile Implementation Plan What to build, in order

1️⃣ Bottom Navigation Bar

Replace sidebar with a fixed bottom nav: Dashboard · Sites · + (FAB) · Labour · Settings. The FAB opens a quick-add sheet (Client, Site, Labour, Purchase).

  • Remove sidebar from layout.tsx
  • Add BottomNav component with 5 items
  • + button is a floating action button with gradient
  • Active state = highlighted icon + text

2️⃣ Dashboard → Site Cards

Replace the current table-based dashboard with a scrollable list of site cards. Each card shows: name, client, status badge, progress bar (budget %), spend summary, overdue amount (if any).

  • Horizontally scrollable stats row at top
  • Tap card → navigate to site detail
  • Overdue payments widget at bottom
  • Pull-to-refresh support

3️⃣ Site Detail → 3 Tabs

Consolidate 5 separate pages into one page with 3 tabs (Overview, Labour, Materials). Each tab is a scrollable section.

  • Back button in header, status badge inline
  • Overview: budget card + quick stats grid + phase list
  • Labour: labour cards with days/rate/paid/balance
  • Materials: stock cards with color-coded % remaining

4️⃣ Bottom Sheets for Actions

All forms use bottom sheets (shadcn Sheet component) instead of full pages: Record Payment, Add Phase, Record Purchase, Add Site, Add Client.

  • Sheet component with side="bottom"
  • Auto-fills context (e.g., balance due)
  • Big buttons, simple inputs
  • Close = dismiss, no navigation

5️⃣ Remove Dead Pages

Cut: Kanban page, Reminders page, Phase detail page, Site financial dashboard page. Their functionality moves into the 3 tabs or the site overview.

  • Delete or archive the route folders
  • Update sidebar nav (now bottom nav)
  • Keep routes for backward compat if needed

6️⃣ Touch & Performance

Mobile-optimize everything: minimum 44px tap targets, no hover states, proper touch scrolling, no pinch-zoom needed.

  • All buttons min-h-[44px]
  • Remove hover: variants, add active: scale in stead
  • Overflow-x-auto for tables with scrollbar hidden
  • Sticky bottom nav, sticky tab bar
🔄 Navigation: Sidebar → Bottom Nav Why this is the right move
❌ Sidebar (current)
  • Hidden behind hamburger on mobile
  • Requires 2 taps to navigate anywhere
  • Covers content when open
  • Hard to reach with thumb
  • 8 nav items = too much choice
✅ Bottom Nav (proposed)
  • Always visible, 1 tap to navigate
  • Thumb-reachable on any phone
  • Doesn't cover content
  • 5 items = focused, no choice overload
  • FAB button for quick actions
UI debt tally: We'll cut ~4 pages (Kanban, Reminders, Phase detail, Site dashboard), remove the sidebar, add a BottomNav component, convert 3 route pages to tab sections, and build 3-4 bottom sheet forms. The site detail page becomes the single most important screen — it's where 90% of work happens.
